# Portrait API Debug Log — 2026-02-27

## The Problem
The live site on Netlify (faithfulfriendportraits.co.za) was broken:
- Gallery showed no images
- OG rich text preview showed `undefined` in the image URL: `https://faithfulfriendportraits.co.za/portraits/undefined`

Both issues traced back to the `/api/portraits` endpoint returning a 500 error on Netlify.

## Root Cause
`server/api/portraits.get.ts` originally used Node.js `readdir` to list files in `public/portraits` at runtime. This works in local dev but fails on Netlify because serverless functions don't have filesystem access to the project's `public/` directory at runtime.

## What We Tried (Did Not Work)

### Attempt 1 — Nitro `serverAssets` with `getKeys()`
- Added `nitro.serverAssets` in `nuxt.config.ts` pointing to `./public/portraits`
- Changed API to use `useStorage('assets:portraits').getKeys()`
- **Result:** Broken in both dev and production. `getKeys()` unreliable for serverAssets.

### Attempt 2 — Build hook + `server-data/` + `getItem()`
- `build:before` hook generates `server-data/portraits.json`
- API uses `useStorage('assets:server-data').getItem('portraits.json')`
- **Result:** Dev worked. Production still broken. `useStorage` via `serverAssets` does not work on Netlify.

### Attempt 3 — Static file `public/portraits-list.json` fetched directly
- `build:before` hook writes to `public/portraits-list.json`
- Page and composable fetch `/portraits-list.json` directly (no API route)
- Gallery worked (client-side fetch). OG preview still broken.
- **Result:** `$fetch('/portraits-list.json')` with a relative URL during SSR fails on Netlify — the serverless function cannot reach its own CDN.

## Final Solution — Bundled JSON Import via API Route ✅

### How it works
The `build:before` hook in `nuxt.config.ts` generates two files at build time:
1. `public/portraits-list.json` — static file on CDN (kept for good measure)
2. `server/portraits-data.json` — statically imported by the API route

`server/api/portraits.get.ts`:
```ts
import list from '../portraits-data.json'
export default defineEventHandler(() => list)
```

The JSON is bundled directly into the Nitro serverless function at compile time.
During SSR, `$fetch('/api/portraits')` is an internal Nitro call — no HTTP, no filesystem, always works.

Both generated files are in `.gitignore`.

### Why it works on Netlify
| Approach | Dev | Netlify |
|---|---|---|
| `readdir` at runtime | ✅ | ❌ no filesystem |
| `useStorage` serverAssets | ❌ | ❌ not bundled |
| `$fetch('/static-file.json')` in SSR | ✅ | ❌ can't reach own CDN |
| Static JSON import in API route | ✅ | ✅ bundled into function |

## Result
- Gallery loads all portrait images ✅
- OG rich text preview shows a random portrait on every request ✅
- WhatsApp preview confirmed working ✅
- Drop-in workflow intact — just add images to `public/portraits/`, commit, push ✅
